# Codex Change Log

Short log for code and architecture changes made by Codex.

Current protected architecture:

```text
stage1 frozen BERT latent autoencoder
stage2 Riemannian metric flow over latent space
FlowNet + MetricNet
natural velocity / metric-aware sampling
```

## 2026-05-08

- Added this change log.
- Rule going forward: before editing, Codex must state what will be added, deleted, modified, whether it conflicts with the current Riemannian architecture, expected impact, and risk.
- Future code changes should add a short entry here.
- Added stage1 decode debug prints in `parallel_decoder.py` for blank `original` / `predicted` output. No architecture impact; this only exposes token ids/tokens when decoding becomes blank.
- Added validation for `bert-base-uncased` tokenizer/config loading in `parallel_decoder.py`. No architecture impact; prevents poisoned local caches that map normal text to `[UNK]` ids like `[2, 1, 1, ...]`.
- Added Riemannian diagnostics in `train_stage2_conditional.py`: metric regularization, metric min/max, initial/raw/generated latent scale, and decoder CE before/after metric-flow sampling. No architecture impact.
- Added Riemannian rollout loss in `train_stage2_conditional.py`, training through short `flow / metric` ODE paths used by inference. No architecture conflict; increases memory/time modestly.
- Added rollout norm preservation loss and raw-norm validation scoring in `train_stage2_conditional.py` to counter latent norm collapse observed during Riemannian rollout training. No architecture conflict; may trade some CE gains for healthier latent scale.
- Changed `MetricNet` log-diagonal bounding from hard clamp to smooth `tanh` in stage2 training and inference, and reduced rollout norm loss weight after norm stabilization. No architecture conflict; restores metric gradients near the bound and softens radial overconstraint.
- Added exact-identity initialization for fresh `MetricNet` heads and a short metric regularization warmup in `train_stage2_conditional.py` after scratch tanh-bound runs showed rapid early metric anisotropy expansion. No architecture conflict; should slow metric gaming while FlowNet learns.
- Added zero initialization for fresh `FlowNet.out_proj` in `train_stage2_conditional.py` after scratch runs showed large step-0 rollout norm spikes from random vector-field initialization. No architecture conflict; starts fresh ODE rollouts at zero velocity before learning.
- Added a separate higher-LR optimizer group and gradient diagnostics for FlowNet self/cross attention gates in `train_stage2_conditional.py` after gate values stayed nearly static. No architecture conflict; reveals whether gates are gradient-starved or just LR-starved.
- Increased fresh FlowNet attention gate initialization and gate optimizer multiplier in `train_stage2_conditional.py` after diagnostics showed nonzero gate gradients but negligible metric impact. No architecture conflict; tests whether gated attention is underpowered or redundant.
- Lengthened the stage2 metric regularization warmup from 500 to 1000 steps in `train_stage2_conditional.py` after repeated gate-gradient drops appeared near the same warmup multiplier. No architecture conflict; delays metric anisotropy release so gates can train longer.
- Increased direct and rollout decoder CE loss weights in `train_stage2_conditional.py` after warmup/gate tuning improved cosine and val score but left raw decoder CE stuck. No architecture conflict; shifts pressure toward decoder-manifold accuracy.
- Restored direct and rollout decoder CE loss weights to `0.05` and added weighted decoder-loss logging in `train_stage2_conditional.py` after higher CE weights inflated total `rloss`, regressed cosine, and caused token-lock repetition. No architecture conflict.
- Made stage2 WikiText loading prefer the local Hugging Face datasets cache before falling back online in `train_stage2_conditional.py`, avoiding startup stalls from remote `HEAD` request timeouts. No architecture impact.
- Increased stage2 rollout training from 4 to 8 ODE steps and reduced rollout batch from 128 to 64 in `train_stage2_conditional.py` after generated samples stayed repetitive despite improving cosine/CE. No architecture conflict; better matches 16-step inference while containing cost.
- Reverted stage2 rollout training to 4 steps with batch 128 in `train_stage2_conditional.py` after the 8-step / batch-64 trial regressed val score, raw CE, and sample quality. No architecture conflict.
- Added fixed seeding, seeded dataloader shuffling/workers, deterministic cuDNN selection, and deterministic validation sampling in `train_stage2_conditional.py` after identical-config runs showed large metric variance. No architecture impact.
- Added configurable persistent DataLoader workers in `train_stage2_conditional.py` to reduce Python multiprocessing `/tmp/pymp-* Directory not empty` cleanup races after validation. No architecture impact.
- Added validation-time argmax token-collapse diagnostics in `train_stage2_conditional.py` after cosine/raw CE improved while samples stayed repetitive. No architecture impact; reports entropy, unique-token ratio, max-token fraction, and dominant tokens for generated vs oracle decodes.
- Added rollout decoder-hidden manifold loss in `train_stage2_conditional.py` after collapse diagnostics showed generated latents produce high-entropy decoder logits with biased argmax repetition. No architecture conflict; stage1 remains frozen while stage2 is nudged toward decoder-internal hidden states.
- Disabled the rollout hidden-loss probe and added rollout oracle-logit KL distillation in `train_stage2_conditional.py` after hidden matching lowered entropy but worsened token collapse. No architecture conflict; stage2 now gets direct pressure to match frozen decoder token-rank distributions from real latents.
- Disabled rollout oracle-logit KL, tightened the metric log bound, added rollout pairwise latent-diversity loss, and included token-collapse penalties in validation scoring in `train_stage2_conditional.py` after KL improved cosine but worsened argmax collapse. No architecture conflict; stage1 remains frozen and stage2 still trains the Riemannian metric flow.
- Changed rollout diversity loss in `train_stage2_conditional.py` from sequence-mean pairwise matching to capped valid-token pairwise matching, and increased its weight after sequence-level diversity reduced metric expansion but did not fix token-basin collapse. No architecture conflict; decoder remains frozen and stage2 still trains the Riemannian metric flow.
- Added optional decoder adaptation in `train_stage2_conditional.py` after flow-only token-level diversity kept metric geometry tight but left generated argmax collapse above threshold. Architecture intentionally changes for this labeled experiment: encoder and decoder BERT body stay frozen, only decoder `project_up` / `to_logits` train with real/generated CE and frozen-teacher preservation KL, saved separately as `stage2_conditional_decoder_adapt_best.pt`.
- Updated `inference_stage2_conditional.py` to auto-prefer `stage2_conditional_decoder_adapt_best.pt` when present and load an adapted decoder from the stage2 checkpoint. Architecture impact matches the labeled decoder-adaptation experiment; preserves fallback to the flow-only checkpoint.
- Stabilized decoder adaptation in `train_stage2_conditional.py` after early adaptation reduced generated CE but oscillated collapse metrics: lowered decoder LR and generated-latent CE weight, increased preservation KL weight, and ramped generated-latent CE over two epochs. Architecture impact remains limited to the labeled decoder-adaptation experiment.
- Added rollout generated-logit balance loss and stronger collapse-aware validation scoring in `train_stage2_conditional.py` after stabilized decoder adaptation improved cosine/raw CE but entered periodic dominant-token collapse cycles. Architecture impact remains limited to the labeled decoder-adaptation experiment; the new loss penalizes batch-wide generated suffix probability concentration without increasing decoder CE.
- Disabled the generated-logit balance training loss in `train_stage2_conditional.py` after it worsened dominant-token collapse, while keeping stronger collapse-aware validation scoring. Added top-k/top-p sampled validation decode beside argmax decode to test whether high-entropy generated logits contain usable distributional text under the biased argmax surface.
- Made sampled top-k/top-p decoding the primary generated readout in stage2 validation logs, renamed argmax output/collapse metrics as diagnostics, and lowered default validation/inference sample temperature from `0.9` to `0.8` for a little more coherence while preserving diversity. No architecture conflict; training losses are unchanged and argmax collapse remains visible in validation scoring.
- Added explicit checkpoint reporting and `--stage1` / `--stage2` CLI arguments to `inference_stage2_conditional.py` after checkpoint selection became ambiguous during model comparisons. No architecture impact; inference now prints resolved checkpoint paths, metadata, and short state fingerprints so different model loads are visible.
- Added validation-time decoder distribution diagnostics in `train_stage2_conditional.py` comparing generated suffix logits against oracle real-latent logits: entropy, top-k mass, batch concentration, special-token mass, and KL to oracle. No architecture impact; this only helps identify whether sampled-output collapse is caused by biased generated-latent decoder distributions.
- Enabled a gentle rollout `KL(oracle || gen)` logit-alignment loss in `train_stage2_conditional.py` after diagnostics showed generated latents decode to overly flat token distributions rather than over-concentrated ones. Reduced rollout diversity pressure and made the KL direction explicit in logs/checkpoint metadata. No architecture conflict; this keeps the Riemannian metric flow and decoder-adaptation experiment unchanged while adding oracle-rank pressure to generated rollout latents.
- Replaced the non-generalizing rollout `KL(oracle || gen)` pressure with a rollout entropy-gap loss in `train_stage2_conditional.py`: KL is disabled, diversity pressure is restored, and generated rollout logits are nudged toward oracle real-latent entropy using the frozen teacher decoder when decoder adaptation is active. No architecture conflict; this targets decoder-confidence/manifold sharpness without forcing exact token-rank matching.
- Changed rollout entropy shaping in `train_stage2_conditional.py` to a one-sided high-entropy penalty with a short schedule after the first entropy-gap run briefly improved top-k mass but then oversharpened/oscillated into a new token attractor. Entropy weight is reduced, diversity pressure is lowered, and entropy shaping now runs full strength for one epoch then decays. No architecture conflict; this keeps the Riemannian metric flow unchanged.
- Changed the one-sided rollout entropy penalty in `train_stage2_conditional.py` from a short pulse to a persistent low-weight guardrail after decay removed the top-k gains by epoch 3. Weight is halved, full-strength duration now spans the 20-epoch run, and decay is disabled. No architecture conflict; this tests whether sustained mild entropy pressure can hold the ep1-2 distribution gains without returning to symmetric oversharpening.
- Changed the rollout entropy penalty in `train_stage2_conditional.py` to decode calibrated generated rollout latents after persistent entgap self-extinguished on raw 4-step rollouts while validation decoded calibrated 16-step latents. No architecture conflict; this tests whether matching the validation latent scale gives the one-sided entropy signal a better proxy for generated distribution sharpness.
- Reverted calibrated rollout latents in the entropy penalty after the calibrated-entgap run collapsed into a sharp `"the"` token attractor and widened generated CE despite improving raw flow CE. The entropy penalty again decodes raw rollout latents, preserving the persistent one-sided entgap configuration without batch-stat calibration gaming. No architecture conflict.
## 2026-05-09
- Added a gated rollout generated-CE loss in `train_stage2_conditional.py` after entropy-only sharpening proved able to make confident wrong-token attractors. The new loss uses the frozen teacher decoder when decoder adaptation is active, applies only to rollout tokens whose generated entropy is above oracle entropy by a margin and whose top-1 probability is below a cap, and logs active fraction/top-1. No architecture conflict; this adds correctness-aware pressure while guarding against single-token overconfidence.
- Lowered the gated rollout generated-CE top-1 cap to `0.25` for the next fresh run and added validation target-correctness diagnostics (`top1_acc` and target-token probability for generated vs oracle logits). No architecture conflict; this checks whether sharper generated distributions are assigning mass to the right target tokens rather than only becoming more confident.
- Added a guarded rollout target-probability loss in `train_stage2_conditional.py` after validation showed generated logits assigning only ~7% probability to the true target token versus ~86% for oracle real-latent logits. No architecture conflict; the new low-weight loss reuses frozen-teacher rollout logits and activates only when target probability is far below oracle target probability while generated top-1 confidence remains below a safety cap.
- Changed stage2 rollout training to a softer 8-step path in `train_stage2_conditional.py` after decoder-side losses improved training-time target probability but did not transfer to 16-step validation sampling. Rollout batch is reduced to 64 and rollout/norm/diversity weights are lowered to test the proxy-gap hypothesis without overconstraining the metric flow. No architecture conflict.
- Split the oversized `train_stage2_conditional.py` into focused stage2 modules: `stage2_config.py`, `stage2_data.py`, `stage2_riemannian.py`, `stage2_losses.py`, and `stage2_eval.py`. No architecture impact; the training entrypoint now keeps orchestration/checkpointing while preserving the same Riemannian metric-flow and decoder-adaptation behavior.

- Removed disabled/failed rollout probe losses, duplicate rollout decoder CE, and a repeated teacher real-logit decode from `train_stage2_conditional.py` / `stage2_losses.py` after RTX 4090 runs were memory-limited to small rollout batches. No architecture conflict; the stage2 Riemannian `FlowNet + MetricNet` path remains intact while decoder-adaptation generated CE now owns the generated-logit CE signal, reducing decoder forward passes and log/checkpoint clutter.
